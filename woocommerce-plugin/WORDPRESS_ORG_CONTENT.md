## 📖 **Full Description** (For WordPress.org plugin page)

```
=== AI Fraud Detection for WooCommerce ===

Protect your WooCommerce store from fraudulent transactions using advanced artificial intelligence and machine learning. Automatically detect fraud in real-time, manually verify suspicious orders, or bulk-check historical transactions with a simple CSV upload.

== The Problem ==

E-commerce fraud costs online merchants billions annually. Chargebacks, lost inventory, and processing fees add up quickly. Traditional rule-based systems miss sophisticated fraud while flagging legitimate customers, hurting your revenue and reputation.

== The Solution ==

AI Fraud Detection for WooCommerce uses advanced machine learning to analyze every transaction with 90%+ accuracy. Get instant fraud predictions with clear explanations of why orders are flagged, so you can make informed decisions and protect your business without losing good customers.

== Key Features ==

**🤖 Automatic Fraud Detection**
Every new order is automatically analyzed in real-time (<200ms). Suspicious transactions are immediately flagged, placed on hold, and you're notified via email - all without any manual intervention.

**🔍 Manual Order Verification**
Not sure about a specific order? Click the "Check for Fraud" button on any order page to get instant AI analysis with confidence scores and risk factors.

**📊 CSV Bulk Upload**
Analyze hundreds of historical transactions at once. Upload a CSV file, track real-time progress, and download detailed fraud reports. Perfect for:
- Reviewing backlog of suspicious orders
- Analyzing seasonal patterns
- Generating fraud reports for stakeholders
- Auditing payment processor flags

**🛡️ Explainable AI**
Every fraud prediction includes the top 3 contributing factors with clear explanations:
- "High transaction amount for new customer"
- "Temporary email domain detected"
- "Shipping address doesn't match billing"

No black box - you always know WHY an order is flagged.

**📧 Email Alerts**
Get instant notifications when fraud is detected:
- Order details and customer info
- Fraud confidence score
- Top risk factors
- Direct link to review order

**⚡ Real-time Processing**
- Predictions in under 200ms (typical: 50-150ms)
- No checkout delays for customers
- Processes after order completion
- Progress bars for bulk operations

**📈 Results Export**
Download fraud analysis results as CSV:
- All checked transactions with verdicts
- Confidence scores and risk factors
- Filter to fraud-only or all results
- Perfect for reporting and auditing

**⚙️ Fully Customizable**
- Adjust fraud threshold (sensitivity)
- Enable/disable automatic checking
- Control order auto-hold behavior
- Toggle email notifications
- Test API connection before going live

== How It Works ==

**Setup (5 minutes):**
1. Install the plugin
2. Configure your fraud detection API endpoint
3. Set your preferred fraud threshold (0.7 recommended)
4. Enable automatic detection
5. Test the connection - done!

**Automatic Mode:**
1. Customer places order on your store
2. Plugin sends order data to AI API (encrypted)
3. ML model analyzes 12+ risk factors in milliseconds
4. Fraud verdict returned with confidence score
5. If fraud detected: order held, you're notified
6. You review and make final decision

**Manual Mode:**
1. Open any WooCommerce order
2. Click "Check for Fraud" in sidebar
3. View instant AI analysis
4. See confidence score and top risk factors
5. Decide to fulfill or cancel

**Bulk Upload Mode:**
1. Prepare CSV with transaction data
2. Upload to WooCommerce → Bulk Check
3. Watch real-time progress
4. Review results dashboard
5. Export report as CSV
```

## ❓ **Comprehensive FAQ**

```markdown
= Is this plugin free? =

Yes! The plugin is 100% free and open source under the MIT license. You can use it, modify it, and distribute it freely. There are no hidden costs, per-transaction fees, or paid upgrades.

= Do I need to pay for the fraud detection API? =

The API is also open source and free. You can:
- Self-host it on your own server (free, but requires technical setup)
- Use cloud hosting like AWS, DigitalOcean, etc. (you pay hosting provider)
- Use a managed service when available (optional, paid)

= How accurate is the fraud detection? =

Our machine learning model achieves:
- 90%+ recall (catches 90% of actual fraud)
- 85%+ precision (85% of flags are real fraud)
- <5% false positive rate

Accuracy improves as you adjust the threshold to match your business patterns.

= Will it slow down my checkout process? =

No! The fraud detection runs AFTER checkout completes, so customers experience zero delay. Typical analysis takes 50-200ms and happens in the background.

= What data is sent to the fraud detection API? =

Only fraud-relevant metadata:
- Order amount and payment method
- Email domain (NOT full email address)
- Customer type (new/returning)
- Billing city and country
- Items count
- Order timestamp

We NEVER send:
- Credit card numbers
- CVV codes
- Full email addresses
- Customer passwords
- Personal messages

= Is it GDPR compliant? =

Yes, when configured properly. The plugin:
- Processes minimal data (data minimization)
- Doesn't store unnecessary personal info
- Can use HTTPS for encrypted transmission
- Provides audit logging
- Follows privacy-by-design principles

= Can I adjust the fraud detection sensitivity? =

Yes! The fraud threshold setting (0.0 to 1.0) controls sensitivity:
- 0.5-0.6: Very sensitive (more alerts, more false positives)
- 0.7: Balanced (recommended for most stores)
- 0.8-0.9: Conservative (fewer alerts, might miss some fraud)

Start with 0.7 and adjust based on your results after 1-2 weeks.

= What if the API is down? =

The plugin has graceful fallback:
- Orders are NOT blocked
- Error is logged for admin review
- You're notified of API issues
- Normal checkout continues

Set up monitoring and alerts for your API to minimize downtime.

= Can I use this with other fraud prevention tools? =

Yes! This plugin works alongside:
- Payment gateway fraud filters
- Google reCAPTCHA
- Other security plugins
- Manual review processes

It adds an additional AI-powered layer of protection.

= Does it work with all payment gateways? =

Yes! The plugin works with any WooCommerce payment gateway:
- Stripe
- PayPal
- Credit card processors
- Cash on delivery
- Bank transfers
- All others

= Can I test it before going live? =

Absolutely! We recommend:
1. Install on staging site first
2. Test with sample orders
3. Use the bulk CSV check on historical data
4. Review accuracy and adjust threshold
5. Then activate on production site

= What if I get false positives? =

False positives are rare (<5%) but can happen:
1. Review the risk factors shown
2. Manually release the order if legitimate
3. Adjust fraud threshold higher (e.g., 0.7 → 0.8)
4. Over time, monitor and fine-tune

The explainable AI helps you understand why orders are flagged.

= How do I handle flagged orders? =

When an order is flagged:
1. Check your email alert
2. Review order in WooCommerce
3. Look at fraud confidence score and risk factors
4. Contact customer if needed (verify phone, address)
5. Decide: fulfill order or cancel as fraud
6. Document decision for future reference

= Can I bulk check historical orders? =

Yes! Use the CSV bulk upload feature:
1. Export orders to CSV from WooCommerce
2. Format according to template (or use download template button)
3. Upload to WooCommerce → Bulk Check (CSV)
4. View results and download fraud report

Perfect for auditing past transactions or seasonal analysis.

= Does it support multiple languages? =

The plugin is translation-ready. We welcome translations! The default language is English, but all text can be translated using WordPress translation tools.

Yes! The MIT license allows:

Perfect for agencies serving multiple e-commerce clients.

We actively maintain the plugin with:
```

## 🎯 **Installation Instructions** (Detailed)

```markdown
= Automatic Installation =

1. Log into your WordPress admin panel
2. Navigate to Plugins → Add New
3. Search for "AI Fraud Detection for WooCommerce"
4. Click "Install Now" on this plugin
5. Click "Activate Plugin"
6. Go to WooCommerce → Fraud Detection
7. Configure your settings
8. Click "Test Connection" to verify
9. Save changes

= Manual Installation =

**Method 1: Upload via WordPress**
1. Download the plugin ZIP file
2. Log into WordPress admin
3. Go to Plugins → Add New → Upload Plugin
4. Choose the downloaded ZIP file
5. Click "Install Now"
6. Click "Activate Plugin"
7. Configure at WooCommerce → Fraud Detection

**Method 2: FTP Upload**
1. Download and extract the ZIP file
2. Upload the 'ai-fraud-detection-woocommerce' folder to '/wp-content/plugins/'
3. Log into WordPress admin
4. Go to Plugins
5. Find "AI Fraud Detection for WooCommerce"
6. Click "Activate"
7. Configure at WooCommerce → Fraud Detection

= Backend API Setup =

The plugin requires the fraud detection API backend. You have two options:

**Option A: Docker (Easiest)**
```
git clone https://github.com/tanveer-ahmed986/fraud_detection_system_ecommerce.git
cd fraud_detection_system_ecommerce
docker compose up -d
```
API available at: http://localhost:8000

**Option B: Manual Install**
```
git clone https://github.com/tanveer-ahmed986/fraud_detection_system_ecommerce.git
cd fraud_detection_system_ecommerce/backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

**Option C: Cloud Hosting**
Deploy the backend to:
- AWS EC2, Lambda, or ECS
- DigitalOcean Droplets or App Platform
- Google Cloud Run
- Heroku
- Any cloud provider

See GitHub repository for detailed deployment guides.

= Configuration =

After installation and activation:

1. **Navigate to Settings**
   - Go to WooCommerce → Fraud Detection

2. **Configure API Endpoint**
   - Enter your API URL (e.g., http://localhost:8000)
   - For cloud: https://your-api-domain.com

3. **Set Fraud Threshold**
   - Recommended: 0.7 (70% confidence)
   - Adjust later based on results

4. **Enable Features**
   - ✓ Automatic Detection (recommended)
   - ✓ Auto-Hold Suspicious Orders (recommended)
   - ✓ Email Alerts (recommended)

5. **Test Connection**
   - Click "Test Connection" button
   - Verify you see "Connection successful!"
   - Check that "Model loaded: Yes"

6. **Save Settings**
   - Click "Save Settings"
   - Configuration complete!

= First Use =

**Test Automatic Detection:**
1. Place a test order on your store
2. Order is automatically checked
3. Check order edit page for fraud analysis
4. Review email if fraud detected

**Test Manual Check:**
1. Go to WooCommerce → Orders
2. Click any existing order
3. Find "🛡️ AI Fraud Detection" box in sidebar
4. Click "Check for Fraud"
5. View instant results

**Test Bulk Upload:**
1. Go to WooCommerce → Bulk Check (CSV)
2. Download CSV template
3. Add sample transaction data
4. Upload and click "Start Fraud Check"
5. View results and export report

= Troubleshooting Installation =

**Plugin won't activate:**
- Check WordPress version (need 5.8+)
- Check WooCommerce is installed and active
- Check PHP version (need 7.4+)

**Can't find settings:**
- Go to WooCommerce (not WordPress Settings)
- Look for "Fraud Detection" submenu
- If missing, deactivate and reactivate plugin

**API connection fails:**
- Verify API is running (visit http://your-api:8000/health)
- Check firewall allows connection
- Ensure endpoint URL is correct (include http://)
- Check WordPress can make outbound requests

**Need help?**
- Check FAQ above
- Visit support forum
- See documentation on GitHub
- Contact developer
```
